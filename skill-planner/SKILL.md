# Skill Planner

## Overview

使用此 skill 在调用 skill-creator 之前，系统化地规划和设计新的 skill。通过引导式问答帮助你：
- 明确 skill 的核心目标和价值
- 定义完整的功能范围
- 设计清晰的使用流程
- 规划配置选项和输入输出
- 确保最佳实践和易用性

**Announce at start:** "我正在使用 skill-planner skill 来帮你系统化地规划新 skill。"

## 核心规划模块

### 1. 需求定义阶段
引导用户回答以下问题：
- Skill 的主要目标是什么？
- 解决什么具体问题？
- 目标用户是谁？
- 与现有 skill 有什么区别？

### 2. 功能设计阶段
- 核心功能列表（按优先级排序）
- 每个功能的详细描述
- 功能之间的依赖关系
- MVP（最小可行产品）范围

### 3. 使用场景设计
- 主要使用场景
- 典型工作流程
- 输入输出示例
- 成功/失败案例

### 4. 配置与交互设计
- 可配置参数
- 用户交互流程
- 提示语设计
- 确认机制

### 5. 最佳实践规划
- 与现有 skill 的集成建议
- 错误处理策略
- 文档规范
- 测试要点

## 规划输出

完成规划后，生成以下内容：

### 1. SKILL.md 草稿
包含完整的：
- name 和 description
- 详细的功能说明
- 使用步骤
- 配置说明
- 最佳实践

### 2. 创建检查清单
- [ ] 功能范围明确且可实现
- [ ] 使用场景清晰且具体
- [ ] 配置项设计合理
- [ ] 文档结构完整
- [ ] 已有类似 skill 分析完成
- [ ] 技术可行性验证

### 3. 实施建议
- 优先级排序
- 分阶段实施计划
- 关键依赖项
- 风险评估

## 使用流程

### 阶段 1: 需求收集
1. 询问 skill 的基本信息（名称、目标）
2. 了解要解决的问题
3. 确认目标用户和使用场景

### 阶段 2: 功能分析
1.  brainstorm 核心功能
2. 按优先级排序
3. 定义 MVP 范围
4. 识别技术挑战

### 阶段 3: 详细设计
1. 设计用户交互流程
2. 定义输入输出格式
3. 规划配置选项
4. 考虑错误处理

### 阶段 4: 文档生成
1. 生成 SKILL.md 草稿
2. 创建检查清单
3. 提供实施建议
4. 确认是否需要调整

完成后，引导用户：
- 查看生成的规划文档
- 根据需要进行调整
- 准备好后调用 skill-creator 创建 skill
