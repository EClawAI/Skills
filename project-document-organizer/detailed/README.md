# 详细文档规模规范

## 覆盖范围
- 项目所有组件、架构、模块、子模块及内部系统
- 所有功能和流程
- 所有相关的业务逻辑

## 详细程度
- 深入分析模块内部实现
- 详细说明类关系、算法流程、接口细节
- 提供完整的实现思路和代码示例
- 确保文档内容全面详细

## 文档结构
- 严格按照树状结构生成完整目录层级
- 为每个子项目创建独立的文件夹目录
- 为每个子模块创建相应的文件夹目录
- 包含所有子模块文档

## 文档类型
- 根目录文档：
  - ARCHITECTURE.md - 项目整体架构文档
  - MODULES_OVERVIEW.md - 总模块文档
  - DIRECTORY_STRUCTURE.md - 目录结构说明
  - SETUP.md - 项目搭建和环境配置
  - FAQ.md - 常见问题解答
  - CHANGELOG.md - 文档修改记录
- 子项目文档：
  - ARCHITECTURE.md - 子项目架构文档
  - MODULES.md - 子项目模块总览
- 模块文档：
  - README.md - 模块概览
  - FLOW.md - 关键流程的流程图
  - UML.md - 类关系图
  - API_REFERENCE.md - 接口说明
  - 详细功能设计文档

## 详细功能设计文档必须包含
- 功能概述：模块的功能描述和定位
- 实现思路：模块的设计思路和实现方案
- 核心类与接口：关键类和接口的说明
- 重点实现代码：核心功能的实现代码或伪代码
- 流程图：关键流程的流程图
- 类关系图：核心类之间的关系图
- 使用说明：模块的使用方法和示例
- 配置说明：模块的配置项和配置方法
- 常见问题：使用过程中可能遇到的问题和解决方案
- 性能考虑：性能优化的考虑和建议
- 依赖关系：模块依赖的其他模块或外部库
- 扩展性考虑：模块的可扩展性设计和未来可能的扩展点
- 测试策略：模块的测试方法和测试用例
- 部署说明：模块的部署方式和注意事项
- 版本历史：模块的版本变更记录
- 维护指南：模块的维护方法和注意事项

## 适用场景
- 深入理解项目
- 代码审查
- 重构前准备
- 新人培训
- 需要全面了解项目实现细节

## 生成要求
- 递归分析每一个文件，按系统或功能归类
- 为每一个系统或功能生成详细功能设计文档
- 生成相关UML类关系图、流程图、时序图
- 确保所有模块都有对应的详细文档
- 检查确认所有相关文档均已生成
- 对未生成的文档进行补全
- 确保文档结构完整，没有缺失的部分
- 验证所有模块都有对应的详细功能设计文档
