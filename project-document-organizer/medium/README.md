# 一般文档规模规范

## 覆盖范围
- 项目整体架构
- 所有核心模块
- 关键业务流程

## 详细程度
- 提供模块级详细说明
- 包含关键流程和接口
- 适当深入模块内部实现
- 平衡详细程度和文档长度

## 文档结构
- 生成根目录文档
- 为核心模块创建子目录
- 包含模块级文档
- 保持清晰的目录层次结构

## 文档类型
- 根目录文档：
  - ARCHITECTURE.md - 项目架构概览
  - DIRECTORY_STRUCTURE.md - 目录结构说明
  - SETUP.md - 项目搭建和环境配置
  - FAQ.md - 常见问题解答
  - CHANGELOG.md - 文档修改记录
- 核心模块文档：
  - MODULES.md - 模块划分和说明
  - API_REFERENCE.md - API接口文档（如适用）
  - DATABASE_SCHEMA.md - 数据库结构（如适用）
  - CONFIGURATION.md - 配置说明

## 适用场景
- 项目交接
- 团队成员快速熟悉项目
- 一般开发需求
- 需要了解核心模块实现

## 生成要求
- 确保覆盖所有核心模块
- 提供足够详细的模块说明
- 包含关键流程的描述
- 保持文档结构清晰
- 避免过于深入的实现细节
