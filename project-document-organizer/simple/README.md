# 简单文档规模规范

## 覆盖范围
- 项目整体架构
- 主要模块概览
- 核心功能简介

## 详细程度
- 仅提供高层级概述
- 不深入模块内部实现
- 重点关注整体结构和核心功能

## 文档结构
- 仅生成根目录下的主要文档
- 不创建子模块目录
- 文档数量控制在基础范围内

## 文档类型
- ARCHITECTURE.md - 项目架构概览
- DIRECTORY_STRUCTURE.md - 目录结构说明
- SETUP.md - 项目搭建和环境配置
- FAQ.md - 常见问题解答
- CHANGELOG.md - 文档修改记录

## 适用场景
- 快速了解项目概览
- 初次接触项目时
- 仅需要了解项目基本情况

## 生成要求
- 确保文档内容简洁明了
- 突出项目的整体架构和核心功能
- 避免过多的细节描述
- 保持文档结构清晰易读
